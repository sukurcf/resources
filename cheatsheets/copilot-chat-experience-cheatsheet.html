Implicit Context (Auto-Included)

  • Active file in the editor is always included
  • Current text selection is sent automatically
  • File name provides language/framework hints
  • In Agent mode, workspace is searched autonomously
  • Open tabs provide additional context signals
⚠️ Content exclusion rules (org-level) can prevent certain files from being sent to the model.

Session Management

/clear Start a fresh chat session
Checkpoints Auto-snapshot before each agent action
Restore Hover request → revert to previous state
Fork Create new branch of conversation
Edit & Resend Click previous prompt to edit & re-run
Parallel Run multiple sessions via + button
💡 Start a new session for unrelated tasks — avoids context pollution.
⚙️

Configure Your Chat Session

Session Type Local, Background (CLI), or Cloud
Agent Ask, Edit, Agent, Plan, or custom agents
Permission Auto-approve, prompt, or restrict tool use
Model GPT-4.1, Sonnet 4.6, Gemini 3.1 Pro, etc.
💡 Auto model selection gives a 10% multiplier discount and excludes models with multiplier > 1×.
🧠

Tips & Best Practices

  • Be specific — state language, framework, expected I/O
  • Break down complex tasks into smaller steps
  • Include expected output — test cases, criteria
  • Ask AI to ask clarifying questions first
  • Iterate — refine with follow-ups, don't rewrite
  • Use /clear for new, unrelated tasks
  • Attach images for visual context (mockups, errors)
  • Course-correct early — steer mid-request if needed

Good vs Bad Prompts

❌ Bad Prompt
"Make this code better"
Vague · no language · no criteria · no expected output
✅ Good Prompt
"Refactor to reduce O(n²) → O(n log n).
Add tests that verify performance."
Specific · measurable · includes verification
