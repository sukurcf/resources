Deep Dive

Every Stage Explained

Detailed breakdown of each step in the Copilot pipeline

💬

Step 1 — User Prompt

The developer triggers Copilot through one of several entry points in their editor or on GitHub.

⌨️
Inline Completions
Ghost text appears as you type — powered by Fill-in-the-Middle (FIM) using both prefix and suffix context
💬
Chat / Ask Mode
Natural language questions in the chat panel — uses #file, #codebase, @workspace for context
🤖
Agent Mode
Autonomous multi-step editing — Copilot determines which files to change, runs terminal commands, iterates on errors
✏️
Edit Mode
You choose the files, Copilot proposes edits — more controlled than Agent mode
🖥️
CLI (gh copilot)
Command-line interface via GitHub CLI — explain commands, suggest fixes, generate shell scripts
🌐
GitHub.com
PR summaries, code review, issue triage, Copilot Spaces — available directly on the platform
📂

Step 2 — Context Gathering (The Prompt Library)

Algorithms in the Copilot extension extract, prioritize, and assemble context into a structured prompt. This is the "prompt engineering" that happens behind the scenes.

📄
Current File (Prefix + Suffix) Code before and after cursor via Fill-in-the-Middle (FIM) paradigm — 10% boost in acceptance rate
📑
Neighboring Tabs All open editor tabs are scanned for matching code patterns — 5% boost in acceptance rate
📋
Custom Instructions .github/copilot-instructions.md, .instructions.md, .copilot/ config files
🧩
Skills & Agents SKILL.md bundles and custom AGENTS.md — domain-specific knowledge packs
🔧
MCP Servers External tool servers providing database access, API calls, file ops, and custom data
🔎
Workspace Indexing Semantic search with embeddings, symbol graph, and codebase-wide retrieval
📌
Copilot Spaces Curated context collections — code, docs, specs organized for specific tasks
🧠
Copilot Memory Learned repo-specific facts that persist across sessions for coding agent & code review
💡 Even a low-quality match from neighboring tabs is better than no context at all. The prompt library includes ALL partial matches to maximize relevance.
🛡️

Step 3 — Pre-Model Filters

Before the prompt reaches the LLM, several safety and policy filters are applied.

1
Responsible AI Content Filter — Blocks harmful, offensive, or inappropriate content from being sent to the model
2
Content Exclusion Rules — Organization admins can exclude specific files/paths via policies (e.g., secrets, proprietary code)
3
Prompt Injection Detection — Scans for attempts to manipulate the model via adversarial inputs in code or comments
4
Policy Enforcement — Enterprise/org policies control which features, models, and capabilities are allowed
5
Scope Validation — Ensures prompts are coding-related; blocks non-coding or off-topic queries
🔀

Step 4 — Copilot Proxy Service

The central routing layer hosted on GitHub/Azure infrastructure.

🔄
Model Routing
Routes to the appropriate model based on user selection or automatic assignment (GPT, Sonnet, Gemini, etc.)
📊
Rate Limiting & Quotas
Manages plan-based usage limits — Free, Pro, Pro+, Business, Enterprise tiers
🔐
Authentication
Validates GitHub tokens, org membership, license entitlements
📡
Telemetry & Logging
Tracks acceptance rates, latency, usage patterns — no code stored by default
⚡
Caching Layer
Optimal caching ensures neighboring tabs and FIM add zero latency
🧠

Step 5 — AI Language Model

The model processes the assembled prompt and generates completions. Multiple models are available — user-selected or auto-routed.

Model Provider Best For Tier
GPT-4o OpenAI General-purpose coding, chat, inline completions All plans
Claude Sonnet Anthropic Complex reasoning, large codebase understanding Pro+ / Biz / Ent
Gemini Google Multi-modal, long context windows Pro+ / Biz / Ent
Grok xAI Fast code generation, creative solutions Pro+ / Biz / Ent
Claude Opus Anthropic Deep analysis, architectural decisions Pro+ / Biz / Ent
BYOK Models Your provider Custom: AWS Bedrock, Azure, OpenAI API, etc. Bring Your Own Key
💡 The model generates n completions in parallel. Post-model filters then select ≤n to show the user, ranked by relevance and quality.
🔍

Step 6 — Post-Model Filters

After the model generates completions, several checks ensure quality and safety before showing results.

1
Public Code Matching — Checks generated code against public GitHub repos. If "block matches" is enabled, matching code is filtered out
2
License Detection — Identifies matching OSS licenses and provides attribution links when matches are found
3
Output Content Filter — Blocks harmful, offensive, or off-topic content in generated responses
4
Quality Ranking — Ranks multiple completions by relevance, correctness, and contextual fit
5
Vulnerability Scanning — Detects common security issues (hardcoded secrets, SQL injection patterns, etc.)
